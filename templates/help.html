RouterCut — 우분투에서 일반 사용자로 돌리기

앱은 로그인한 사용자(예: kw)로 실행합니다. root로 전체 앱을 띄울 필요는 없고, SMB 마운트만 sudo로 허용하면 됩니다.

1. 패키지 설치

sudo apt update
sudo apt install -y python3 python3-pip python3-venv cifs-utils

프로젝트 폴더에서 pip install -r requirements.txt (또는 venv 사용).

2. mount / umount 를 비밀번호 없이 허용 (sudoers)

아래에서 kw를 본인 계정 이름으로 바꿉니다.

sudo visudo -f /etc/sudoers.d/routercut

파일에 한 줄 추가:

kw ALL=(root) NOPASSWD: /usr/bin/mount, /usr/bin/umount, /bin/mount, /bin/umount

저장 후 권한: sudo chmod 440 /etc/sudoers.d/routercut (visudo가 보통 맞춰 줌).

3. 앱 실행 환경 변수

마운트 시 sudo -n을 쓰려면:

export ROUTERCUT_MOUNT_USE_SUDO=1

SMB 계정은 웹 화면 「SMB 로그인 설정」에서 저장합니다. 로컬 DB 파일(data/routercut.db)에만 들어갑니다.

특수 도메인이 필요하면 서버에서만 ROUTERCUT_SMB_DOMAIN 환경 변수를 쓸 수 있습니다(보통 불필요).

4. 포트와 디버그

기본 포트는 15777입니다. 바꾸려면:

export PORT=15777

디버그(자동 리로드)는 기본 꺼짐. 켤 때만:

export ROUTERCUT_DEBUG=1

5. 실행 방법

포그라운드(터미널에서 테스트):

cd /path/to/routercut
export ROUTERCUT_MOUNT_USE_SUDO=1
python3 app.py

브라우저: http://본인PC_IP:15777

백그라운드:

cd /path/to/routercut
./scripts/run-background.sh

로그 기본: /tmp/routercut.log · PID: /tmp/routercut.pid
