RouterCut — 동적 마운트(CIFS) + 로컬 DB

장비 등록이 “안 된다”면

  • 같은 IP 중복이면 서버가 409를 돌려줍니다. 메시지: 「이미 같은 IP로…」 → 기존 행을 고치거나 IP를 바꾸세요.
  • 저장 위치: data/routercut.db (프로젝트 폴더). 디스크 풀·권한·다른 프로세스 잠금을 확인하세요. WAL 모드 사용 중입니다.

동적 마운트 동작

장비가 SMB 공유면, 스캔·이미지 요청 시에만 mount -t cifs //IP/공유 … 로 붙였다가 작업이 끝나면 umount 합니다. 상시 마운트가 아닙니다.

커널 마운트는 보통 root 권한이 필요합니다. 앱은 일반 사용자로 두고, 아래처럼 mount/umount 만 sudo 로 허용하는 방식을 권장합니다.

1. 패키지

sudo apt update
sudo apt install -y python3 python3-pip cifs-utils
cd /path/to/routercut
pip install -r requirements.txt

2. sudo (NOPASSWD) 예시

kw 를 본인 계정으로 바꿉니다.

sudo visudo -f /etc/sudoers.d/routercut
kw ALL=(root) NOPASSWD: /usr/bin/mount, /usr/bin/umount, /bin/mount, /bin/umount

실행 전:

export ROUTERCUT_MOUNT_USE_SUDO=1

scripts/run-background.sh 는 기본으로 이 값을 켭니다.

3. SMB 계정

웹 「SMB 로그인 설정」 → data/routercut.db 의 app_settings 에만 저장됩니다.

4. 실행 · 포트

export ROUTERCUT_MOUNT_USE_SUDO=1
python3 app.py

기본 포트 15777. 디버그는 ROUTERCUT_DEBUG=1 일 때만.

5. SMB 경로

Windows \\10.56.164.91\Result2 → IP 10.56.164.91, 공유 이름 Result2. 공유 루트 바로 아래 yyyyMMdd/Result.csv 구조.
