RouterCut — 우분투에서 일반 사용자로 돌리기

왜 예전에 400 / mount 오류가 났나

리눅스에서 mount -t cifs 는 커널에 “이 SMB 공유를 내 폴더에 붙여라”라고 요청하는 것이라, 보통 root 권한(CAP_SYS_ADMIN)이 필요합니다. 일반 유저로 앱만 돌리면 “permission denied”, “mount system call failed” 류 메시지와 함께 스캔 API가 400을 내기 쉽습니다.

지금 버전은 마운트를 쓰지 않습니다. Python 패키지 smbprotocol으로 SMB 프로토콜(445포트)만 직접 연결해 Result.csv·이미지를 읽습니다. 그래서 sudo 없이 같은 유저 계정으로 실행하면 됩니다.

1. 패키지 설치

sudo apt update
sudo apt install -y python3 python3-pip python3-venv

cifs-utils는 더 이상 필수가 아닙니다(마운트 안 함).

cd /path/to/routercut
pip install -r requirements.txt

가상환경(python3 -m venv .venv) 쓰는 것을 권장합니다.

2. 방화벽 · 네트워크

PC에서 장비(또는 파일 서버)로 TCP 445가 열려 있어야 합니다. 장비 쪽 Windows 방화벽에서 “파일 및 프린터 공유” 허용 여부도 확인하세요.

3. SMB 계정

웹 화면 「SMB 로그인 설정」에 사용자·비밀번호를 넣습니다. 저장 위치는 이 PC의 data/routercut.db 뿐입니다.

일부 장비만 서명을 강제하면 터미널에서 export ROUTERCUT_SMB_REQUIRE_SIGNING=1 후 앱을 다시 실행해 볼 수 있습니다(기본은 끔에 가깝게 두어 호환 우선).

4. 포트·디버그

앱 기본 포트 15777. 바꾸려면 export PORT=원하는포트.

디버그(리로더)는 기본 꺼짐. 필요 시만 export ROUTERCUT_DEBUG=1.

5. 실행

cd /path/to/routercut
python3 app.py

브라우저: http://이_PC_IP:15777

백그라운드:

./scripts/run-background.sh

로그: /tmp/routercut.log · PID: /tmp/routercut.pid

6. SMB 경로

Windows \\10.56.164.91\Result2 와 같이 쓰면, 앱에는 IP 10.56.164.91, 공유 이름 Result2 로만 넣습니다. 그 아래에 yyyyMMdd/Result.csv 구조가 있다고 봅니다.
